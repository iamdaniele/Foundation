Method Detail
{Object} json(url, type, options)
Prepares a JSON/JSONP request using the specified method. This method internally proxies Titanium.HTTPClient, so further scrambling can be made, which means that:
  • This method performs a request and parses the result to get a valid JSON object, keeping the original response.
  • This method can perform a JSONP request with a specified callback name, or use the default one.
This method will return a proxy to an instance of Titanium.HTTPClient, which means that you will still be able to use the same (documented) entry points you should expect to find on that object.
Parameters:
{String} url
The request URL
{String} type Optional, Default: "GET"
The type of request. So far, GET and POST are been tested successfully. Titanium should support PUT and DELETE, although these methods haven't been tested.
{Object} options Optional
A dictionary of options specific to Foundation.Request
{String} options.user Optional
Username for HTTP basic authentication
{String} options.pass Optional
Password for HTTP basic authentication
{Boolean} options.authentication Optional, Default: false
Enables or disabled basic authentication for the current request. If true, you must provide username and password.
{Boolean|String} options.jsonCallback Optional
True to use Foundation's internal callback (Foundation.Request.callback). You can provide a function name if you need a different callback. The function must accept the response body as parameter.
Returns:
{Object} A proxy to Ti's HTTPClient. You should expect to use the same entry points as it.
